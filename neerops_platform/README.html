📋 Project Overview

NEEROps v9.0 is a production-grade autonomous DevOps platform that achieves Level 4 autonomy through goal-centric orchestration. It implements a sophisticated 10-layer event-driven architecture that autonomously manages the complete software delivery lifecycle from pull request to production deployment.

🎯 Goal-Centric

Defines DevOps outcomes (latency, uptime, cost) as goals rather than tasks. System optimizes for these goals autonomously.

🧠 Intelligent Solver Selection

Deterministic-first hierarchy: Heuristics (90%) → RL (9%) → LLM (1%), reducing costs and improving reliability.

🔒 Safety-First Design

8 critical safety constraints enforced at state transitions. No invalid decisions can reach production.

⚙️ Event-Driven Architecture

All 10 layers communicate through Redis Streams EventBus, enabling scalability and layer independence.

🏗️ System Architecture

Three Core Goals (G1-G3)

G1: Response Time ⚡

Latency P99 < 100ms. Measured across all customer-facing endpoints.

G2: Availability 🟢

Uptime > 99.95% (SLA). Error rate < 0.1% across production.

G3: Cost Efficiency 💰

TCO optimization. Reduce infrastructure costs by 30% YoY while maintaining SLA.

Event-Driven Communication

All inter-layer communication flows through Redis Streams EventBus:

  • Layer publishes event with PR context
  • Consumer group processes asynchronously
  • Result returned to orchestrator via event
  • No direct function calls between layers
  • Complete auditability and replay capability

🔟 The 10 Autonomous Layers

L0: Cognition
Solver selection hierarchy. Chooses optimal approach for each decision.
L1: Understanding
Builds Knowledge Graph with IaC drift detection and risk assessment.
L2: Review
5-gate security pipeline: Gitleaks → Semgrep → Mutmut → Pact → Intent
L3: Build
7-step CI pipeline: merge → install → test → docker → trivy → cosign → push
L4: Deploy
Bayesian canary with Beta-Binomial gates. 5% → 50% → 100% rollout.
L5: Monitor
3-tier anomaly detection: threshold → 3-sigma → Bayesian.
L6: Healing
5-tier escalation: Heuristic → RL → Embedding → Local → LLM.
L7: Feedback
RL training. 3D reward: Safety (0.5x) + Uptime (0.3x) + Cost (0.2x)
L8: Reasoning
World Model with Monte Carlo. Pre-computes top-3 healing plans.
L9: Meta-Learning
System evolution: prompt evolution, model selection, threshold optimization.
⚡ Event Flow: Each layer listens for events, processes asynchronously, and publishes results. This decoupling enables parallel development, testing, and deployment of individual layers without affecting others.

✨ Key Features

🎯 Safety Constraints (S1-S8)

Enforced at Orchestrator pre-transition to prevent invalid states:

  • S1: Deployment requires successful L2 review
  • S2: Canary must pass Bayesian gate
  • S3: Rollback on L5 anomaly detection
  • S4: Healing must pass dry-run validation
  • S5: Vault lock during deployment
  • S6: No repo commits during healing
  • S7: Action deduplication (prevent circular loops)
  • S8: Human approval for critical actions

🤖 ML Components

  • Heuristic Library: Rule storage with semantic search
  • Embedding Cache: pgvector-based semantic similarity
  • RL Training: 7-day rolling window daily training
  • Prompt Evolution: Thompson sampling on 5 variants
  • Model Selection: Weekly benchmarks with auto-promotion

🔐 Security Pipeline

  • Gitleaks: Secret detection (hard block)
  • Semgrep: SAST scanning (blocks on HIGH)
  • Trivy: Container CVE scanning (CRITICAL/HIGH blocks)
  • ZAP: DAST on staging deployment
  • Falco: Runtime security monitoring

📊 Orchestrator Features

  • 12-state PR FSM with full history tracking
  • 10 circuit breakers (one per layer)
  • Redis SETNX locks for concurrent PR handling
  • OTEL tracing integration
  • QLDB audit logging

🚀 Quick Start

Option 1: Local Python (< 5 minutes)

python3 -m venv venv source venv/bin/activate pip install -r requirements.txt python tests.py # Run 25+ tests python cli.py start # Start platform

Option 2: Docker Compose (< 10 minutes)

docker build -t neerops:latest . docker-compose up -d # Starts all services docker-compose logs -f neerops

Option 3: Kubernetes (< 20 minutes)

docker tag neerops your-registry/neerops:v9.0.0 docker push your-registry/neerops:v9.0.0 kubectl apply -f kubernetes/deployment.yaml kubectl get pods -n neerops
